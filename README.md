# TRPG Log Archive

Roll20 등 TRPG 채팅 로그를 **백업**하고 **시네마틱하게 감상**하는 정적 웹사이트입니다.
외부 AI API 없이, 브라우저 안에서 정규식으로 로그를 블록으로 변환합니다.
모든 데이터는 이 깃허브 레포지토리(`TRPG_Log`) 안의 파일로 관리됩니다.

---

## 1. 폴더 구조

```
TRPG_Log/
├─ index.html              # 진입점
├─ css/
│  ├─ base.css             # 디자인 토큰 · 공통 · 상단 바
│  ├─ library.css          # 홈(서가) · 카드 · 장면 목차
│  ├─ editor.css           # 등록/편집기
│  └─ viewer.css           # 시네마틱 뷰어
├─ js/
│  ├─ util.js              # 공용 유틸 · 이미지 경로 · 아바타
│  ├─ parser.js            # Roll20 로그 파서(정규식)
│  ├─ data.js              # 데이터 로딩(manifest+세션) + 샘플 폴백
│  ├─ library.js           # 홈/목차 뷰
│  ├─ viewer.js            # 뷰어 + 인라인 BGM
│  ├─ editor.js            # 등록/편집
│  └─ app.js               # 초기화/라우팅
├─ data/
│  ├─ manifest.json        # 화면에 표시할 세션 목록
│  └─ sessions/            # 세션별 JSON 파일
│     ├─ ses_wall.json
│     └─ ses_twodays.json
├─ image/                  # 캐릭터/커버 이미지
│  └─ <세션명>/<캐릭터>.png
└─ README.md
```

---

## 2. 배포 (GitHub Pages)

1. 이 폴더 전체를 `TRPG_Log` 레포지토리에 올립니다.
2. 레포지토리 **Settings → Pages → Branch: main / root** 로 설정합니다.
3. `https://<사용자명>.github.io/TRPG_Log/` 로 접속합니다.

> 데이터(`data/*.json`)는 `fetch` 로 읽기 때문에 **웹 서버(GitHub Pages)** 에서 동작합니다.
> 로컬에서 `index.html` 을 파일로 직접 열면 데이터 로드가 막혀 **샘플 데이터**가 표시됩니다.
> 로컬 확인이 필요하면 폴더에서 `python -m http.server` 후 `http://localhost:8000` 으로 여세요.

## 3. 캐릭터 / 커버 이미지 올리기

- 이미지는 `image/<세션명>/<파일명>.png` 형태로 올립니다.
  - 예: `image/성벽을 넘어서/에스티니앙.png`
  - 세션 카드(썸네일)용 커버는 보통 `image/<세션명>/_cover.png` 로 둡니다.
- 편집기의 캐릭터 **이미지 경로** 칸과 **세션 카드 이미지 경로** 칸에 이 경로를 그대로 적습니다.
  - `TRPG_Log/image/...` 처럼 레포명을 붙여 적어도 자동으로 정리됩니다.
- **이미지가 없는 캐릭터는 글자 없이 단색 원**으로 표시됩니다. (색상은 캐릭터 색상 사용)

## 4. 로그 등록 / 편집 → 페이지에 반영하는 흐름

1. 상단 **＋ 로그 등록** (또는 세션/장면에서 **✎ 편집**) 으로 편집기를 엽니다.
2. **폴더·세션 정보** 를 채웁니다.
   - 기존 폴더는 드롭다운에서 선택, 새 폴더는 옆 칸에 직접 입력하면 새로 생성됩니다.
3. **등장인물** 을 추가합니다. 역할을 고릅니다.
   - **KPC / NPC → 뷰어 왼쪽**, **PC → 뷰어 오른쪽** 에 말풍선이 배치됩니다.
   - 여기 등록하지 않은 이름은 모두 **NPC** 로 처리됩니다.
   - 상단 등장인물 소개란에는 **PC·KPC만** 표시됩니다.
4. **장면** 을 추가하고, 장면을 선택한 뒤 **로그를 붙여넣고 [변환]** 합니다.
   - 붙여넣을 때 이미지(`<img>`)는 자동 제거되고 볼드/기울임만 유지됩니다.
   - 블록은 사이사이의 **＋ 여기에 블록 추가** 로 중간 삽입할 수 있고,
     각 블록에서 **유형 변경 / 텍스트 볼드·기울임(B·I) / 순서 이동 / 삭제** 가 가능합니다.
   - 추가 블록 유형: 일반 나레이션, 강조 나레이션, BGM(유튜브 URL + 제목), 대사, 주사위.
5. **⬇ 세션 JSON 다운로드** 를 누르면 `<세션id>.json` 파일이 받아집니다.
6. 받은 파일을 `data/sessions/` 에 올립니다.
7. `data/manifest.json` 의 목록에 경로를 추가합니다.

```json
{
  "sessions": [
    "data/sessions/ses_wall.json",
    "data/sessions/ses_twodays.json",
    "data/sessions/내가_올린_세션.json"
  ]
}
```

8. 커밋하면 페이지 새로고침 시 반영됩니다.

> 편집기의 **미리보기** 버튼은 저장 없이 현재 장면을 즉석에서 확인하는 용도입니다.

## 5. 파서가 인식하는 로그 패턴

| 입력 형태 | 변환 결과 |
|---|---|
| `이름: "대사" 지문 "대사"` | 대사 블록 (따옴표 안=대사, 밖=지문) |
| `:서술 문장` (맨 앞 콜론) | 일반 나레이션 |
| GM 화자 줄 | 강조 나레이션 |
| 이름 → 항목 → 등급 → 굴림 → `vs.` → 목표치 (줄바꿈형) | 주사위(대상치) |
| `…기준치:65/32/13굴림:47판정결과:보통 성공` (한 줄) | 주사위(기준치) |
| `rolling 1d100` → `(` → `80` → `)` → `=` → `80` | 주사위(단순 굴림) |

변환 후에는 편집기에서 유형·내용을 자유롭게 수정할 수 있습니다.

## 6. 뷰어 사용

- 세션 카드를 누르면 **장면 목차**가 뜨고, 장면을 고르면 시네마틱 뷰어가 열립니다.
- **BGM 블록**은 로그 중간에 놓이며, 독자가 직접 ▶ 를 눌러 재생합니다.
  - 자동 반복되며, 다른 BGM 블록을 누르면 이전 곡이 페이드아웃된 뒤 새 곡이 재생됩니다.
- 우상단 **◑** 로 라이트/다크 테마를 전환할 수 있습니다.

> 폰트(Noto Serif KR / Pretendard)와 유튜브 BGM은 인터넷 연결이 있어야 로드됩니다.
> 브라우저 자동재생 정책상 BGM은 ▶ 를 한 번 눌러 시작합니다.
